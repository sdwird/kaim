Engine Architecture

kaim isn't just an aimbot, it's a meticulously optimized Lua engine built to bypass common execution limits while maintaining flawless frame rates.

01

Zero-Bridge Drawing Proxy

Most scripts update the C++ Drawing API (`Drawing.new`) every single frame, causing severe memory bridging latency and inevitable stutters.

Our custom `ND()` proxy acts as an intermediary. It caches all properties locally in Lua and only dispatches updates to the C++ engine when a value actually changes. Result: 90% reduction in C++ bridge overhead.

02

Raycast Allocation Caching

"Smart Mode" checks up to 6 different body parts per player per frame. Standard aimbots instantiate a new `RaycastParams` filter array for each part, flooding the garbage collector.

Kaim caches the `FilterDescendantsInstances` array based on the `_lastVisChar`. It reuses the exact same C++ allocation across all body part checks for that character, cutting allocation costs by 6x per player.

03

Dynamic Target HUD

4 unique layouts (Premium, Competitive, Compact, Minimalist). Instead of generating new drawing objects when you swap styles, it securely recycles the exact same 15 drawing objects.

The HUD leverages the proxy's `__index` fallback to safely query `TextBounds` from the underlying engine, ensuring flawless pixel-perfect right alignment without expensive string-math.

04

Heartbeat Hitbox Bypass

Nested loops checking thousands of parts on Heartbeat will instantly tank FPS.

Kaim implements a strict `_hbActive` state flag. When the Hitbox Expander is disabled, the script permanently halts all mathematical operations regarding hitboxes, executing zero code on Heartbeat cycles until you turn it back on.
